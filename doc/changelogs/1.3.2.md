# Changelog - 1.3.2 (Patch Release)

Release date: 2026-03-04

Summary:
- Patch-level stability and polish after v1.3.0. This release includes the version bump from 1.3.0 -> 1.3.2, documentation updates, and a set of robustness and UI improvements detected across the codebase.

Detailed changes (cataloged across project files):

1) Packaging & metadata
- `manifest.json`: version bumped to `1.3.2` to match this release.
- `src/Plugin.cs`: `BepInPlugin` attribute updated to `"1.3.2"` and startup log updated to reference v1.3.2.
- `CHANGELOG.md` & `README.md`: headers updated to include 1.3.2 and a short patch-note entry was added.

2) Release automation scripts
- `scripts/ManifestUpdater.csx`, `scripts/PatchPluginFile.csx`, `scripts/ReadmeCreator.csx`, `scripts/ChangelogCreator.csx`, `scripts/Release.bat` remain present and used for manifest and packaging automation. Note: `PatchPluginFile.csx` writes BepInPlugin with package id `org.wicked.cheat_menu` (script behavior unchanged in this patch).

3) GUI and UX improvements
- `src/gui/CheatMenuGui.cs`:
  - New top-level quick action button: `Unlock EVERYTHING` (calls `CombatDefinitions.UnlockAbsolutelyEverything()` with user feedback notifications).
  - Farming category: added `Clear Berry Bushes` quick action invoking `CultUtils.ClearBerryBushes()`.
  - Resources category: item quantity slider moved into UI and supports controller adjustment with acceleration; `ResourceDefinitions.ItemSpawnQty` bound to the slider.
  - Controller navigation: improvements to selection, visual hover style, auto-scroll logic, persistent hint bar positioning (now above the menu during animation), and better suppression of R3 double-action.
  - `GiveAllItems()` behavior includes more DLC-safe filters and reporting about skipped DLC items.

- `src/gui/GUIUtils.cs`:
  - Theme and styles consolidated. New fonts selection and refined GUIStyle creation for buttons, selected/button styles and title bar.
  - Added `CustomWindowScrollableLocked` to support animated windows.

4) Core utilities & DLC safety
- `src/CultUtils.cs`:
  - Added/expanded DLC helper `IsDlcContentName()` which centralizes DLC keyword checks used across unlock/give operations.
  - `ClearBerryBushes()` implemented to remove berry/bush type structures safely from base.
  - Numerous defensive helpers and improved null-safety across follower, inventory and ritual utilities.

5) Definitions and feature wiring
- `src/definitions/CombatDefinitions.cs`:
  - New `UnlockAbsolutelyEverything()` implementation (respects DLC ownership and uses several safe unlock steps). Also contains map reveal patches and unlimited ammo/relics features.

- `src/definitions/CultDefinitions.cs`:
  - `UnlockAllStructures()`, `GiveAllClothing()` and other cult-related cheats updated to respect DLC ownership and to update only loaded followers when applying outfits.
  - `ChangeAllRituals()` UI helper and other ritual tooling included.

- `src/definitions/ResourceDefinitions.cs`:
  - `GiveAllItems()` expanded with refined DLC checks and logging for added/skipped items.
  - Inventory helpers and many "Give" cheats were included/extended to use the `ItemSpawnQty` slider.

- `src/definitions/FarmingDefinitions.cs`:
  - Added spawn and management cheats for Woolhaven animals and ranch operations (spawn goat/cow/llama/turtle/crab/spider/snail), with RequiresDLC annotations.

- `src/definitions/DlcDefinitions.cs`:
  - Multiple DLC-specific give functions (seeds, drinks, forge materials, collectibles) and helpers for DLC-only operations.

6) Stability & crash protection
- `src/GlobalPatches.cs`:
  - Added many Harmony patches and defensive prefixes/finalizers: safer `Follower.Init` validation/fallback, `Follower.Tick` guard, `Spine.Skin.AddSkin` null-safety, `Finalizer_Follower_Init` to suppress and recover from crashes during initialization, and `Prefix_PlayerFarming_Bleat` (R3 suppression).
  - These patches aim to prevent loading hangs and NRE spam discovered in earlier releases.

7) Misc utilities & helpers
- `src/CheatConfig.cs` and `src/gui/GUImanager`-related code unchanged in purpose but integrated with the improved UI flows and controller support.
- Several helper classes (`CheatUtils.cs`, `AsyncHelper.cs`, `ReflectionHelper.cs`, `TextureHelper.cs`, `RewiredInputHelper.cs`) support the features and remain present.

Notes & next steps:
- This patch focused on packaging, documentation, and a set of small polish/stability changes. A number of functional improvements (e.g., animal halos, friendly wolf, DLC-safe give/unlock logic, follower init crash recovery, UI improvements) were verified in the codebase and documented above.
- The release automation scripts are present in `scripts/` and can be used to create packages and update manifest/plugin files for future releases. Review `PatchPluginFile.csx` expectations (it targets `org.wicked.cheat_menu`) before relying on it.
